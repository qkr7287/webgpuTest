# AI 3D Commander - 프로젝트 소개

## 이게 뭔가요?

**채팅으로 3D 물체를 만드는 프로그램**입니다.

"빨간 공 만들어"라고 입력하면 화면에 진짜 빨간 공이 나타납니다.
"위로 올려", "회전시켜", "파랗게 바꿔" 같은 한국어 명령도 알아듣습니다.

서버가 필요 없습니다. **브라우저 하나면 끝**입니다.
AI가 내 컴퓨터의 GPU에서 직접 돌아갑니다.

## 어떻게 동작하나요?

```
내가 하는 말        AI가 하는 일           화면에 보이는 것
─────────────    ──────────────────    ─────────────────
"빨간 공 만들어"  →  말을 기계 명령으로 변환  →  빨간 공이 나타남
"회전시켜"       →  회전 명령 생성         →  공이 빙글빙글 돌기 시작
"다 지워"        →  삭제 명령 생성         →  화면이 깨끗해짐
```

핵심은 **AI가 사람 말을 이해해서 3D 명령으로 바꿔주는 것**입니다.
프로그래밍을 몰라도 말로 3D 장면을 만들 수 있습니다.

## 할 수 있는 것들

| 명령 예시 | 결과 |
|----------|------|
| "빨간 공 만들어" | 빨간 구가 생성됨 |
| "파란 상자를 옆에 놓아줘" | 파란 정육면체가 옆에 배치됨 |
| "위로 올려" | 마지막 물체가 위로 이동 |
| "전부 회전시켜" | 모든 물체가 빙글빙글 회전 |
| "조명을 따뜻하게 바꿔" | 조명 색이 주황빛으로 변경 |
| "다 지워" | 화면 초기화 |

만들 수 있는 도형: 상자, 구, 원기둥, 원뿔, 도넛(토러스)
애니메이션: 회전, 통통 튀기, 둥둥 떠다니기

## 기술 스택

TypeScript + Vite + Three.js(WebGL) + WebLLM(WebGPU) + Web Worker

## AI 모델 비교

이 프로그램은 3가지 AI 모델 중 하나를 골라서 사용합니다.
모델이 클수록 똑똑하지만, 그래픽카드 메모리(VRAM)를 많이 먹고 느려집니다.

### Qwen3 1.7B (Fast)

| 항목 | 내용 |
|------|------|
| 크기 | 1.7B (17억 개 파라미터) |
| 필요 VRAM | ~1.5GB |
| 로딩 시간 | 빠름 (수십 초) |

**장점**
- 로딩이 빠르다. 기다림이 적다.
- 가벼워서 일반 노트북에서도 돌아간다.
- 내장 그래픽(iGPU)에서도 시도 가능하다.
- "빨간 공 만들어" 같은 단순 명령은 잘 처리한다.

**단점**
- 복잡한 문장을 잘못 이해할 때가 있다.
- "초록 원뿔을 왼쪽 위에 놓고 회전시켜" 같은 복합 명령에서 실수할 수 있다.
- 맥락(이전 대화)을 기억하는 능력이 약하다.

**추천 상황**: 데모 시연, 빠른 테스트, 저사양 PC

---

### Qwen3 4B (Balanced)

| 항목 | 내용 |
|------|------|
| 크기 | 4B (40억 개 파라미터) |
| 필요 VRAM | ~3GB |
| 로딩 시간 | 보통 (1~2분) |

**장점**
- 복합 명령도 꽤 잘 처리한다.
- 이전 대화를 참고해서 "그걸 올려" 같은 표현을 이해한다.
- 속도와 품질의 균형이 좋다.

**단점**
- 내장 그래픽에서는 무리일 수 있다.
- 1.7B보다 로딩이 느리다.
- 아주 복잡한 장면 설명은 여전히 틀릴 수 있다.

**추천 상황**: 실제 사용, 중급 그래픽카드 보유 시 (GTX 1660 이상)

---

### Qwen3 8B (Best)

| 항목 | 내용 |
|------|------|
| 크기 | 8B (80억 개 파라미터) |
| 필요 VRAM | ~5GB |
| 로딩 시간 | 느림 (2~5분) |

**장점**
- 가장 똑똑하다. 복잡한 문장도 정확히 이해한다.
- "빨간 공 3개를 삼각형으로 배치하고 전부 떠다니게 해" 같은 복합 명령도 처리 가능.
- 맥락 파악 능력이 가장 좋다.
- JSON 명령 형식을 가장 안정적으로 출력한다.

**단점**
- VRAM 5GB 이상의 그래픽카드가 필요하다.
- 첫 로딩이 오래 걸린다 (모델 다운로드 포함).
- 응답 생성 속도가 상대적으로 느리다.

**추천 상황**: 품질 중시, 고사양 GPU 보유 시 (RTX 3060 이상)

---

### 모델 선택 가이드

```
내 그래픽카드 VRAM이...

2GB 이하     →  1.7B (이것만 가능)
3~4GB       →  1.7B 또는 4B
5GB 이상     →  8B 추천, 나머지도 가능
모르겠다     →  1.7B부터 시작해서 올려보기
```

> VRAM 확인법: 작업 관리자(Ctrl+Shift+Esc) -> 성능 탭 -> GPU -> 전용 GPU 메모리

## 이 기술을 어디에 쓸 수 있을까?

### 1. 교육

| 활용 | 설명 |
|------|------|
| 3D 모델링 입문 | 코딩 없이 말로 3D를 체험. 진입 장벽이 사라진다. |
| 수학/과학 수업 | "반지름 2인 구를 만들어" -> 기하학 시각화 |
| 코딩 교육 | AI가 만든 JSON 명령을 보면서 프로그래밍 개념 학습 |

### 2. 디자인/기획

| 활용 | 설명 |
|------|------|
| 빠른 프로토타이핑 | 아이디어를 말로 설명하면 바로 3D로 확인 |
| 공간 레이아웃 | "상자를 왼쪽에, 원기둥을 오른쪽에" -> 배치 검토 |
| 비개발자 협업 | 디자이너/기획자가 직접 3D 장면을 만들어서 소통 |

### 3. 게임/엔터테인먼트

| 활용 | 설명 |
|------|------|
| 레벨 에디터 | 말로 게임 맵 구성 요소 배치 |
| 인터랙티브 콘텐츠 | 관객이 채팅으로 3D 장면을 조작하는 라이브 |
| 어린이용 창작 도구 | 글을 못 써도 말로 세계를 만드는 경험 |

### 4. 접근성

| 활용 | 설명 |
|------|------|
| 시각장애인 보조 | 음성 입력 연동 시 말로 3D 콘텐츠 제작 가능 |
| 운동 제한 사용자 | 마우스/키보드 대신 자연어로 조작 |

### 5. 비즈니스

| 활용 | 설명 |
|------|------|
| 제품 시각화 | "빨간 상자 옆에 파란 원기둥" -> 제품 배치 시뮬레이션 |
| 전시/인테리어 | 가구 배치를 말로 지시하며 레이아웃 확인 |
| 프레젠테이션 | 발표 중 실시간으로 3D 장면 생성 (데모 효과) |

## 기술적으로 특별한 점

**서버가 없다** — AI가 내 브라우저에서 직접 돌아갑니다.
개인정보가 외부로 나가지 않고, 인터넷이 끊겨도 (모델 다운로드 후) 동작합니다.

**GPU를 둘로 나눠 쓴다** — AI 연산은 WebGPU, 3D 그래픽은 WebGL.
같은 그래픽카드지만 서로 다른 통로를 써서 충돌이 없습니다.

**AI가 멈춰도 3D는 멈추지 않는다** — AI는 별도 공간(Web Worker)에서 돌아가서,
AI가 생각하는 동안에도 3D 화면은 부드럽게 60fps로 움직입니다.
